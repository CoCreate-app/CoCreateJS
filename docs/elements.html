CoCreateJS

Learn how to convert your html into a dynamic realtime application.

Collaborative Elements

We have 2 collections by default “modules” and “module_activities”. If the data-collection="" is left empty or if the attribute is not used it will default to “module_activities”.

<div data-collection='module_activities'></div>

If your logic requires a new collection just add it to the attribute and it will be created if it does not exist. 

<div data-collection='custom collection name'> </div>

Forms

Using your forms with CoCreateJS Realtime CRUD is super easy, Form is used for grouping elements and applying attributes dynamically to it childeren. form tag can be left empty or you can apply attributes you would like to apply to childeren elements. attributtes will only be applied to childeren if the attribute does not already exist. If form realtime false it will require submitBtn

You will have the option to set realtime as true or false. If data-realtime set to false then the form will require a button to save.

<button class="submitBtn"> </button>

CRUD Attributes

  • data-realtime boolean optional

    Save data onChange, if false a btn required to save. If attribute is empty or not used Default's to true.

  • data-collection string required

    Defines a collection name where your documents will be stored. sql equivalent is a table.

  • data-document_id string required

    Defines the document by id where your data will be stored. if left empty one will be generated. sql equvalient is a row

  • name string required

    Defines the name in the document containing the value. sql equvalient is a column

  • data-read_value boolean optional

    Fetch value on page load or when updated. If attribute is empty or not used Default's to true.

  • data-save_value boolean optional

    Save data onChange/submitBtn. If attribute is empty or not used Default's to true.

  • data-cursors boolean optional

    Show collaborators cursor positions. If attribute is empty or not used Default's to true.

  • contenteditable boolean optional

    htmltags can be editable and display will collaborators cursors.

  • data-namespace string optional

    Define a unique namespace where messages and changes are sent and received

  • data-room string optional

    Define a unique room where messages and changes are sent and received

  • data-broadcast string optional

    Broadcast messages and changes to other collaborators in the same namespace and room. If attribute is empty or not used default's to true.

  • data-broadcast_sender string optional

    Broadcast messages and changes to self. If attribute is empty or not used default's to false.
